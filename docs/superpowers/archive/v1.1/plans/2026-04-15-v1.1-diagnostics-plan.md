# V1.1 Diagnostics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expand the V1 diagnostics surface into an operator-facing console that exposes auth health, provider health, capability probe outcomes, and policy-aware routing visibility.

**Architecture:** This plan builds on the V1 request lifecycle and V1.1 provider inventory/auth work. It should avoid duplicating raw logs in the UI and instead provide structured diagnostics summaries and drill-downs derived from the runtime and persistence layers.

**Tech Stack:** Rust, Rusqlite, Serde, React 19, TypeScript, `@tanstack/react-query`, `@tanstack/react-table`, existing logs page and tray/runtime summaries, Vitest, Testing Library.
