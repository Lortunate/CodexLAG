# V1.1 Provider Registry Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace provider-specific branching with a shared provider registry that supports OpenAI official and generic OpenAI-compatible providers through one normalized adapter contract.

**Architecture:** V1 currently separates official and relay paths more than V1.1 can afford. This plan introduces a provider registry, shared adapter contracts, and normalized capability/error/account shapes so runtime routing, account inventory, and diagnostics read from one provider model.

**Tech Stack:** Rust, Serde, Tokio, reqwest, existing `providers/*`, routing candidate builder, Rusqlite.
