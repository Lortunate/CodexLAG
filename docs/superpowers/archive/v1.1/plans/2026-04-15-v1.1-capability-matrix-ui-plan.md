# V1.1 Capability Matrix UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the V1 card-style capability presentation with a richer model-level capability matrix using normalized provider inventory and a real data-grid surface.

**Architecture:** The matrix should be a first-class operator tool rather than a decorative table. It should use normalized inventory data from the backend, React Query for fetch lifecycle and invalidation, and TanStack Table for sorting, filtering, column visibility, and comparison-friendly rendering.

**Tech Stack:** React 19, TypeScript, `@tanstack/react-query`, `@tanstack/react-table`, Zod, existing app shell and page primitives, Vitest, Testing Library.
