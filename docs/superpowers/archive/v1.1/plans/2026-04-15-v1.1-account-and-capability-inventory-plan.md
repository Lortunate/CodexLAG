# V1.1 Account And Capability Inventory Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Project provider-backed accounts, sessions, models, and capability data into one normalized runtime inventory consumed by UI, routing, and diagnostics.

**Architecture:** V1.1 must stop teaching the UI and routing layers about provider-specific storage details. This plan adds a unified inventory service that joins persisted account records, provider sessions, capability probes, and manual generic-provider models into one runtime-facing read model.

**Tech Stack:** Rust, Serde, Rusqlite, existing commands/state/runtime modules, provider registry, routing candidates.
