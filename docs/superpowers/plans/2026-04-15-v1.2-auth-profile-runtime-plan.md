# V1.2 Auth Profile Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the V1.1 auth foundation into a provider-neutral desktop auth runtime that supports only the CLIProxyAPI-aligned account login shapes for V1.2: browser OAuth with callback handling and static API key providers.

**Architecture:** The runtime should keep the current OpenAI browser auth flow intact, but promote auth shape to a stable contract that can also describe API-key-backed providers. Browser callback handling remains local and desktop-owned. No device-code flow, custom-protocol-first flow, or remote auth broker should be added in this release.

**Tech Stack:** Rust, Tauri v2, Tokio, Axum, Serde, Rusqlite, reqwest, `openidconnect-rs`, existing secret storage and provider session persistence.
