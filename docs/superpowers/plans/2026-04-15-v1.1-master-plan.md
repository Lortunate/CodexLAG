# CodexLAG V1.1 Master Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver CodexLAG V1.1 with OpenAI in-app login, a shared provider registry, richer capability and diagnostics surfaces, and an archived V1 document set.

**Architecture:** V1.1 keeps the Rust-owned runtime and React/Tauri desktop split from V1, but adds a cleaner platform layer for auth, provider registration, capability inventory, and operator observability. Execution should default to parallel work across isolated git worktrees after the archive move and interface contracts are locked.

**Tech Stack:** Tauri v2, Rust, Tokio, Axum, Serde, Rusqlite, reqwest, `openidconnect-rs`, `@tauri-apps/plugin-opener`, React 19, TypeScript, `@tanstack/react-query`, `@tanstack/react-table`, Zod, Vitest, Testing Library.
