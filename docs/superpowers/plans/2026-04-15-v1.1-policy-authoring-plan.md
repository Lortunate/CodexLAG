# V1.1 Policy Authoring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade policy authoring from minimum V1 editing to a clearer operator workflow with stronger validation, ordering controls, and candidate preview support.

**Architecture:** V1.1 should not replace the persisted routing-policy model, but it should provide a better editing surface on top of it. This plan uses Zod-backed form validation, richer ordering controls, and preview-oriented UI fed by normalized inventory so operators can understand the consequences of their policy changes before saving.

**Tech Stack:** React 19, TypeScript, React Hook Form, Zod, existing policy commands and runtime routing contracts, Vitest, Testing Library.
