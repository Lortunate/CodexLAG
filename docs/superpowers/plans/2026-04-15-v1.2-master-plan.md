# CodexLAG V1.2 Master Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver CodexLAG V1.2 as a bounded desktop-local multi-provider release that adds additional official providers, first-class auth profiles, and a provider-aware operator surface.

**Architecture:** V1.2 keeps the single-machine Tauri desktop architecture from V1 and V1.1, but formalizes auth profiles, extends the built-in provider registry, and upgrades the operator UI so mixed provider families remain understandable and debuggable.

**Tech Stack:** Tauri v2, Rust, Tokio, Axum, Serde, Rusqlite, reqwest, `openidconnect-rs`, desktop protocol/callback handling, React 19, TypeScript, `@tanstack/react-query`, `@tanstack/react-table`, Zod, Vitest, Testing Library.
