# V1.1 OpenAI Auth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add OpenAI official in-app login to the Tauri desktop app using browser launch, a loopback callback, PKCE, persisted session state, automatic refresh, and explicit logout.

**Architecture:** The auth flow should live in a dedicated Rust auth runtime rather than in scattered commands. A temporary callback listener receives the authorization code, the auth runtime exchanges the code through an OpenID Connect/OAuth client, secret material is persisted through the existing secret-storage layer, and UI-facing commands expose session state and login actions.

**Tech Stack:** Rust, Tokio, reqwest, `openidconnect-rs`, Tauri v2, `@tauri-apps/plugin-opener`, Rusqlite, existing secret-store abstraction.
