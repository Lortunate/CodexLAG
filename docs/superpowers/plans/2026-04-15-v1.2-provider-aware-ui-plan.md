# V1.2 Provider-Aware UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the operator-facing frontend around the V1.2 provider set so onboarding, inventory, policy preview, and diagnostics remain coherent when providers use different auth shapes.

**Architecture:** The UI should keep the V1.1 shell and data-layer foundations, but become provider-aware. It needs to render auth profile, degraded health, capability scope, and request explainability without inventing remote-control-plane behavior or provider-specific one-off screens.

**Tech Stack:** React 19, TypeScript, `@tanstack/react-query`, `@tanstack/react-table`, Zod, Vitest, Testing Library, existing Tauri command surface.
