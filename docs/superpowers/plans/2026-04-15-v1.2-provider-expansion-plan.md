# V1.2 Provider Expansion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expand the built-in provider registry for V1.2 to support the bounded official provider set and preserve normalized inventory behavior across OpenAI, Claude, Gemini, and the existing generic OpenAI-compatible provider.

**Architecture:** The registry stays static and built-in. V1.2 adds provider descriptors, adapter implementations, and inventory normalization for the official provider set aligned with CLIProxyAPI config vocabulary. This is not a plugin framework and does not add arbitrary external provider loading.

**Tech Stack:** Rust, reqwest, provider registry and inventory modules, existing persistence and capability probing foundations.
