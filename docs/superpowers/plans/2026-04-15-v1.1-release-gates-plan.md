# V1.1 Release Gates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the V1 verification surface so V1.1 auth, provider, UI, and diagnostics workstreams are all covered by repeatable local and CI release gates.

**Architecture:** This plan preserves the V1 release gates and layers V1.1-specific tests on top of them. The CI workflow should remain simple and deterministic: install frontend dependencies once, run frontend tests, run backend tests, and include the new V1.1 auth/provider/diagnostics suites in the Windows matrix.

**Tech Stack:** GitHub Actions (Windows), Bun, Rust/Cargo, Vitest, existing release workflow and test suites.
