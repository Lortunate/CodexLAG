## Design Context

### Users
CodexLAG is used primarily by programmers operating a local desktop control plane for accounts, relays, keys, policies, and runtime diagnostics. They use it while actively configuring providers, inspecting routing behavior, and debugging local gateway state on their own machine. The interface should support focused, information-dense desktop use without feeling visually noisy.

### Brand Personality
The product should feel concise, minimal, and programmer-friendly. It should communicate precision, control, and clarity rather than marketing polish or decorative flair.

### Aesthetic Direction
The visual direction is a restrained dark operator workbench. The UI should avoid gradients and avoid flashy consumer styling. It should emphasize strong hierarchy, crisp surfaces, clear state grouping, and dense but readable information layout suited for technical users.

### Design Principles
1. Prioritize operator clarity over ornament: state, routing health, auth profile, and diagnostics should be easy to scan.
2. Keep the interface dark, calm, and precise: use restrained contrast, tinted neutrals, and solid colors only.
3. Treat the app like a workbench, not a marketing site: layout should support long sessions and dense information review.
4. Strengthen hierarchy through typography, spacing, and grouping rather than decorative effects.
5. Preserve trustworthiness: degraded states, provider differences, and system consequences must remain explicit.
